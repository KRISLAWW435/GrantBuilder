# 🚀 Инструкция по выгрузке сайта на GitHub Pages

Мы полностью исправили структуру проекта, вернув стандартный, полноценный билд Vite. Ограничивающий и ломающий функции сайта билд в одну папку `docs` был удален. Теперь сайт собирается со всеми необходимыми динамическими чанками (это абсолютно необходимо для корректной работы библиотек вроде `jspdf`, `docx` и других интерактивных сервисов).

Мы настроили **GitHub Actions** (автоматический сборщик). Теперь всё, что вам нужно сделать, — выгрузить исходный код в репозиторий на GitHub. GitHub сам соберёт сайт без ошибок и опубликует его за вас!

---

## 🛠 Шаг 1. Выгрузите код на GitHub

Загрузите весь проект (все файлы из корневой папки) в ваш репозиторий на GitHub. 
*Благодаря файлу `.gitignore` лишние тяжелые папки `node_modules/` и `dist/` не попадут в репозиторий, а загрузится только чистый исходный код.*

Вы можете сделать это через:
- **GitHub Desktop** (самый простой способ): просто перетащите папку проекта в приложение, сделайте комит и нажмите "Publish repository".
- **Командную строку (Git CLI)**:
  ```bash
  git init
  git add .
  git commit -m "Initial commit with auto-deploy workflow"
  git branch -M main
  git remote add origin https://github.com/ВАШ_НИКНЕЙМ/ИМЯ_РЕПОЗИТОРИЯ.git
  git push -u origin main
  ```

---

## 🤖 Шаг 2. Автоматическая сборка (происходит сама)

После того как вы отправите код на GitHub:
1. Перейдите во вкладку **Actions** в панели вашего репозитория.
2. Вы увидите запущенный процесс сборки под названием `Deploy to GitHub Pages`.
3. Сборщик автоматически установит зависимости, скомпилирует весь сайт без ошибок и создаст специальную ветку **`gh-pages`** с готовым кодом для публикации.

---

## ⚙️ Шаг 3. Включите отображение сайта в настройках GitHub

Когда автоматический процесс во вкладке **Actions** завершится (загорится зеленая галочка ✅):

1. Перейдите в **Settings** (Настройки) вашего репозитория на GitHub.
2. В левом вертикальном меню выберите вкладку **Pages**.
3. В разделе **Build and deployment** -> **Source** выберите: `Deploy from a branch` (выбран по умолчанию).
4. В подразделе **Branch** вместо `None` выберите ветку **`gh-pages`** и папку **`/ (root)`**.
5. Нажмите кнопку **Save** (Сохранить).

---

## 🎉 Шаг 4. Готово!

Через 1–2 минуты страница обновится, и наверху появится ссылка на ваш живой сайт:
`https://ВАШ_НИКНЕЙМ.github.io/ИМЯ_РЕПОЗИТОРИЯ/`

Все интерактивные элементы, скачивание списков, генерация файлов и анимации будут работать **абсолютно без ошибок и на 100% идентично тому, как это работает на хостинге разработчика**.
